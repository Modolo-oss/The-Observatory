Data dari real observatory—yang bisa diakses lewat API keys—adalah data hasil transaksi nyata dan performa integrasi Gateway dalam sistem observatory kamu. Berikut gambaran jenis data real yang relevan dan bisa dipakai sebagai API endpoint:

Jenis Data Real Observatory untuk API Keys
Transaction Metrics

Total dan detail setiap transaksi yang dikirim dari observatory ke Gateway/Solana (timestamp, tx hash, success/failure).​

Success rate (persentase transaksi berhasil/gagal).

Error codes dan minimal log event khusus (jika terjadi transaksi gagal).

Performance Metrics

Latency (berapa lama waktu proses satu transaksi dari submit ke confirmation).

Average cost per transaction (SOL/fee yang terpakai dalam benchmark).

Comparison data: Latency dan biaya via Gateway vs RPC tradisional.

Benchmark Results

Hasil run benchmark terakhir (misal: 30 transaksi). Ada data tabel: success rate, latency, fee, throughput.

Rekap summary: min/avg/max latency, min/avg/max fee.

Configuration/Route Status

(Jika bisa) Informasi config aktif untuk Gateway yang dipakai (versi, endpoint, region).

Kapan API Keys Layak Di-Improve
Kalau data seperti di atas benar-benar sudah tersedia di database observatory (bukan mock/hardcoded), API Keys bisa di-"improve" agar developer bisa akses data realtime/terakhir lewat endpoint.​

Kalau belum ada pipeline atau semuanya masih mockup, fitur sebaiknya skip.

Jadi, data real observatory = hasil transaksi, performa, benchmark nyata, dan statistik dari database observatory yang sudah terisi hasil live run. API Keys dipakai buat akses data ini secara programatis oleh team/dev lain/automasi.
